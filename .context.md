# tools — Developer Context

This file is excluded from Cloudflare deployment. It is for developer reference only.

## Project Overview

A collection of browser-based utility tools served as static assets via Cloudflare Workers. Each tool is a **single-file** `index.html` with no build step, no npm, no bundler. Shared logic lives in `utils/`.

Deployed at: `https://tools.bluejar.org/`
GitHub: `https://github.com/humor4fun/tools`
License: AGPL-3.0

## How to Add a New Tool

1. Create `tools/<toolname>/index.html` (single file, no dependencies outside `utils/`)
2. Add a tile in `tools/index.html` (follow existing tile pattern)
3. Add a row in `tools/README.md`
4. Load shared utils (see canonical load order below)
5. Call `renderHeader({title, toolIconSvg, githubPath})` in init
6. Call `initSidebarSections(storageKey, sectionIds[])` if the tool has a sidebar

## How Shared Utils Work

All utils are plain JS globals loaded via `<script src="../utils/x.js">`. No modules, no bundler — works from `file://` and Cloudflare Workers alike.

CSS utils self-load their companion `.css` file on first script execution.

### Canonical Load Order

```html
<link rel="stylesheet" href="../utils/tokens.css">
<!-- tool's own <style> (minus extracted CSS) -->
<script src="../utils/debounce.js"></script>
<script src="../utils/toast.js"></script>
<script src="../utils/clipboard.js"></script>
<script src="../utils/sidebar.js"></script>
<script src="../utils/keyboard.js"></script>
<script src="../utils/color.js"></script>   <!-- barcode/qr only -->
<script src="../utils/slider.js"></script>  <!-- barcode/qr only -->
<script src="../utils/banner.js"></script>  <!-- list only -->
<script src="../utils/undo.js"></script>    <!-- list only -->
<script src="../utils/tabs.js"></script>
<script src="../utils/header.js"></script>
<script> /* tool code */ </script>
```

See `utils/README.md` for full API docs.

## Design System

Always use `tokens.css`. Never hard-code colors or font stacks that are already in tokens.

Key tokens:
- `--bg`, `--bg-sidebar`, `--bg-hover`, `--bg-active` — backgrounds
- `--border`, `--border-light` — borders
- `--text`, `--text-muted`, `--text-dim`, `--text-heading` — text
- `--accent-blue`, `--accent-green` — accents; `--accent` = alias for `--accent-blue`
- `--radius` / `--r` — border radius (6px)
- `--font-mono`, `--font-sans` / `--font-ui` — typefaces

Shared structural pattern:
- `#app` CSS grid: `header (full width) | sidebar (290px) + main (1fr) | statusbar (full width)`
- `header.js` generates: `#hdr-logo`, `.hdr-sep`, `#hdr-title`, `#tab-bar`, `#header-actions`, `#hdr-gh`
- Sidebar: `<div class="sb-sec" id="sec-foo"><div class="sb-hdr">…<span class="sb-chevron">…</span></div><div class="sb-body">…</div></div>`
- Status bar: `<div id="stbar">` with `<span id="stats">` or similar

## Coding Conventions

- Vanilla JS only — no frameworks, no npm, no build step
- `'use strict'` in every `<script>`
- CSS custom properties (tokens) for all colors/spacing
- localStorage keys: `<toolname>_prefs` for general prefs, `<toolname>_tabs` for tab state, `<toolname>_prefs_collapsed` for sidebar collapse (managed by sidebar.js)
- Git history uses descriptive commit messages matching the change scope

## text (formerly mdweb) — Migration Status

`text/index.html` has been migrated to shared utils as follows:

| Item | Status | Notes |
|---|---|---|
| `tokens.css` | ✅ done | inline `:root` colors/fonts/radius removed |
| `toast.js` | ✅ done | inline `showToast()` removed, all call sites updated |
| `header.js` | ✅ done | `renderHeader()` called at script start; `#btn-toggle-log` preserved via `#header-actions`; workspace chips use `#tab-bar` |
| `sidebar.js` | N/A | sidebar is a custom file-tree browser, not the `.sb-sec` pattern |
| `tabs.js` | N/A | workspace tab system is deeply coupled to file tree and auto-refresh; not compatible with generic `initTabs()` |

Remaining tool-specific CSS in `text/index.html`: `.path-chip`, `.ws-chip-*`, `#btn-toggle-log`, badge colours, file-type tokens, layout tokens (`--sidebar-w`, `--header-h`, `--toolbar-h`).

## How to Add a New Util

1. Create `utils/<name>.js` (and `utils/<name>.css` if needed)
2. Expose a single `window.*` global
3. If it has CSS, self-load it from `document.currentScript.src`
4. Document in `utils/README.md`
5. Add to the canonical load order above
6. Add to `wrangler.jsonc` `assets.exclude` if it's a non-deployable doc file
